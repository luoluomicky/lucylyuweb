抛弃传统 PPT，用 HTML 和 AI 打造"永不跑版"的跨国提案

出海笔记系列 #13 · 2026.06

刚刚结束了一场与法国水下成像设备厂商的 Teams 会议。开场切屏的时候，我半开玩笑地跟法国的 CEO 坦承了一句："老实说，这是我第一次抛弃传统办公软件，直接用 HTML 网页代码做了一份演示文稿，我不确定待会儿在跨国会议里跑起来会不会翻车。"

结果出人意料。整个 Pitch 讲下来，不仅没有任何排版错乱，对方 CEO 反而对这种极其干净、不受任何多余干扰的呈现方式感到惊喜。"整个展现极其干练，演讲逻辑非常清晰，Impressive。"一小时后，我把打包好的 HTML 文件发了过去。很快收到了他的邮件反馈："我刚打开了这个演示文件；完美运行（it works perfectly）。"

传统 PPT 的"熵增"陷阱

只要你做过哪怕一次面向海外大客户的 Pitch Deck，就一定经历过被传统办公软件按在地上摩擦的绝望。排版维度的降级、牵一发而动全身的灾难、跨国协作的字体水土不服、文本溢出引发的雪崩——这种极度消耗心智的内耗，本质上是因为传统PPT的排版单位是"手"，你永远在依靠极其不靠谱的肉眼和鼠标拖拽去寻找一种对齐的幻觉。

HTML 的排版单位是极其冷酷且精准的"像素"。前者的每一次修改都在增加系统的混乱度；后者则是用代码将规则写死，精确到小数点。

注入灵魂——让 AI Agent 接管"排版与逻辑"

但我一行代码都没自己敲。我只负责输出最核心的商业大纲和底层逻辑，剩下的排版苦力活，全部甩给了 Claude Code。现在的"低熵"解法，是彻底将形式与内容剥离：我只负责"想"，AI 负责"画"和"讲"。

这个 HTML 演示文稿的底层骨架，其实来源于我的一个业余小项目——一个用来自动生成小红书图文卡片的 Carousel 代码。因为它原本就是用来做社交媒体图文的，所以底层自带了极其方便的"导出 PNG"功能。

一次配置，永久复用

用 HTML 做 PPT 的最大好处不是"好看"，而是"一次配置，永久复用"。品牌色、字体、Logo、页脚——全部写死在 CSS 变量里。老板说"把所有页面的 Logo 换成带 Slogan 的新版"？改一行 CSS 变量，14 页幻灯片同步更新。不需要打开任何演示软件。不需要肉眼对齐。不需要怀疑是不是漏了一页。

而且，HTML 文件的体积小到不可思议——整个演示文件只有几十 KB，可以直接用邮件发送，对方双击即可在浏览器中打开。不需要安装任何软件。字体永远不会回退成 Times New Roman。排版永远不会跑偏。
